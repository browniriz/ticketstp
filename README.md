# Ticketsbot — Telegram Mini App для заявок

Mini App для подачи и обработки заявок. Стек как в проекте Техноточка:
**Google Apps Script (бэкенд) + `index.html` на GitHub Pages + Telegram-бот.**

- Сотрудники создают заявки (Тип, Город → Офис, Имя, Описание) и видят «Мои заявки» со статусом.
- Админы видят все активные заявки в реальном времени, берут в работу (серверный таймер), ставят на паузу, завершают; есть вкладка «История».
- Бот шлёт уведомления в рабочий чат при создании / взятии в работу / решении заявки.

## Файлы

| Файл | Что это |
|------|---------|
| `Code.gs` | Бэкенд Apps Script: один POST-эндпоинт, работа с таблицей, таймер, уведомления. |
| `index.html` | Mini App (UI). Один файл — заливается на GitHub Pages. |

## Развёртывание (по шагам)

### 1. Google Таблица + Apps Script
1. Создай новую Google Таблицу (название любое, напр. «Заявки»).
2. **Расширения → Apps Script**. Удали содержимое `Код.gs`, вставь весь `Code.gs` из этого репо.
3. В блоке `CONFIG` вверху заполни:
   - `BOOTSTRAP_ADMIN_IDS` — твой Telegram tg_id (узнать у [@userinfobot](https://t.me/userinfobot)). Можно несколько через запятую.
   - `NOTIFY_CHAT_ID` — позже (см. шаг 4). Пока можно оставить пустым.
   - `BOT_TOKEN` — уже вписан.
4. Запусти функцию **`setup`** (выбери её в выпадающем списке → Run). Разреши доступы при запросе. Это создаст листы **роли** и **заявки** и добавит тебя админом.

### 2. Публикация веб-приложения
1. **Deploy → New deployment → тип «Web app»**.
2. **Execute as: Me**, **Who has access: Anyone** (обязательно, иначе CORS).
3. Скопируй **Web app URL**.
4. Вставь его в `index.html` → константа `API_URL`.

> ⚠️ После любых правок `Code.gs` нужно снова **Deploy → New deployment** (новый, не «Manage»), иначе изменения не публикуются.

### 3. GitHub Pages
1. Создай репозиторий, залей `index.html`.
2. **Settings → Pages** → ветка `main`, папка `/root`. Дождись деплоя в Actions (1–2 мин).
3. Скопируй URL страницы (вида `https://username.github.io/repo/`).

### 4. Telegram-бот и уведомления
1. У [@BotFather](https://t.me/BotFather): `/setmenubutton` (или Bot Settings → Menu Button) → вставь URL GitHub Pages как Mini App.
2. **Уведомления в чат:** добавь бота в рабочий чат. Напиши там любое сообщение. Открой
   `https://api.telegram.org/bot<ВАШ_ТОКЕН>/getUpdates`
   и возьми `chat.id` (для группы — отрицательное число). Впиши его в `NOTIFY_CHAT_ID` в `Code.gs` и **сделай новый деплой**.

### 5. Роли
- Сотрудники добавлять не обязательно — кто не в листе «роли», тот сотрудник по умолчанию.
- Чтобы добавить ещё админа: впиши строку в лист **роли**: `tg_id | имя | админ`.

## Структура таблицы

**роли**: `tg_id` · `имя` · `роль` (`сотрудник` | `админ`)

**заявки**: `номер` · `дата_создания` · `тип` · `город` · `офис` · `имя_отправителя` · `описание` · `статус` · `tg_id_создателя` · `админ_tg_id` · `админ_имя` · `work_started_at` · `накоплено_сек` · `дата_решения` · `последнее_изменение`

Статусы: `создана` (🔴) → `в работе` (🟡) ⇄ `на паузе` → `решена` (🟢).
Номер заявки: заглавная латинская буква + 3 цифры (напр. `B047`).

## Локальная проверка UI

`index.html` открывается и в обычном браузере (без Telegram) — будет режим «сотрудник».
Полный функционал (роль, заявки) работает только после шагов 1–2 и заполнения `API_URL`.

## Частые проблемы
- **Запросы падают 404 / ничего не грузится** → пустой или старый `API_URL` в `index.html`.
- **CORS / Failed to fetch** → в деплое Apps Script не выставлено «Who has access: Anyone».
- **Изменения Code.gs не применились** → нужен *новый* деплой, а не обновление существующего.
- **Старая версия на Pages** → подожди деплой в Actions и обнови с `Ctrl+Shift+R`.
- **Нет уведомлений в чате** → не заполнен `NOTIFY_CHAT_ID` или бот не в чате.
